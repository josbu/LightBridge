# 升级内容 / Changelog

本文档记录 LightBridge 各版本的升级内容。最新版本在最前。

> 各版本的发布说明同时写入对应的 Git tag 注释中，GitHub Release 页面与本文档保持一致。

---

## v0.3.0

### 新增

- 新增独立的“功能注册中心”，统一管理核心、可选和扩展功能的渐进式注册状态。
- 非核心功能支持启用、停用和继承默认三态控制，并展示依赖、运行组件、错误和重启要求。
- 控制面决策统一作用于路由、菜单、后台 worker 与实际请求路径。

### 调整

- 移除用户可见的模块市场、`/admin/modules` 页面和管理员模块管理 API。
- 保留 LightBridge 托管 Provider 的内部运行时、自动安装、签名校验和远程 UI contribution。
- `marketplace_*` 配置键仅作为内部 Provider registry 的向后兼容键保留。

### 修复

- 修复 `module_runtime` 启动失败时管理入口被同步隐藏、管理员看不到错误原因的问题。
- 修复渐进式控制状态与支付、风控、隐私过滤、运维监控等旧业务开关可能不一致的问题。

完整说明见 `docs/RELEASE_NOTES_v0.3.0.md`。

## v0.1.7

### 新增

- **Custom Provider 支持**：新增独立的 Custom 平台类型，允许配置自定义上游端点 (Base URL) + 协议类型。支持 5 种协议：
  - OpenAI Responses
  - OpenAI Chat Completions  
  - OpenAI Embeddings
  - Anthropic Messages
  - Gemini
- Custom 账户可加入任意分组，按其 protocol 与请求的入站 endpoint 自动匹配调度。
- **Base URL 安全增强**：OpenAI 和 Anthropic 官方账户的 Base URL 现已锁定为官方地址，不再允许自定义（提升安全性）。Gemini Base URL 保持可编辑。

### 优化

- **数据模型优化**：
  - 新增 `accounts.extra[“protocol”]` 字段存储 Custom 账户的协议类型
  - Account DTO 新增 `protocol` 字段用于前端展示和编辑
  - 错误透传规则、配额平台白名单、兜底模型配置均已支持 Custom 平台
- **调度优化**：Custom 账户不受分组类型限制，可被任意分组按协议匹配调度
- **路由优化**：Custom 分组请求根据入站 endpoint 自动分派到对应的原生协议 handler
- 版本控制页面精简：移除”发布版本”区块多余的标题与描述，移除每个版本下方重复的 `LightBridge X.X.X` 名称行。
- 版本控制页面为每个版本新增”升级内容”按钮，可直接查看该版本的升级说明（无需先安装）。

### 数据迁移

- 自动将存量带非官方 Base URL 的 OpenAI/Anthropic API Key 账户迁移为 Custom 平台
- 保留原分组成员关系，实现零破坏迁移
- 更新 `user_platform_quotas` 表约束以支持 Custom 平台

### 修复

- 修复版本升级时同一个账号会被重复迁移为 Gemini OAuth 的问题。
- 修复版本升级时已删除的账户会被重新添加的问题。

### 测试

- 更新测试用例以适配 Base URL 锁定和 Custom 平台
- 所有相关单元测试通过验证

## v0.1.6

### 新增

- 账号导入支持多选 JSON/TXT 文件。
- 账号导入支持上传 ZIP，并自动解压其中可导入的 JSON/TXT 文件。
- 账号导入支持多个远程下载链接，每行一个。
- 新增导入进度条，显示当前导入项与整体进度。
- 导入时支持批量选择分组和批量覆盖账号设置。
- 分组选择支持所有平台，不再仅限 OpenAI。
- 标准导入流程支持 CPA/Codex JSON，包括 `type: "codex"` 数据。
- 新增兼容性模式，可从非标准 JSON/TXT 中提取 token 字段导入。
- API 调用返回给客户端的错误信息支持国际化（i18n），覆盖 502/503/429 等上游错误及无可用账号、并发限流、请求校验等所有网关错误信息，支持中文与英文。
- 新增 `language` 配置项（默认 `en`），用于设置错误信息的默认语言；单个请求可通过 `Accept-Language` 请求头（如 `zh-CN`）覆盖默认语言。

### 修复

- 修复 ZIP 导入 CPA Codex 文件时报 `unsupported data type: codex` 的问题。
- 修复远程导入失败时错误信息过于笼统的问题，现在会显示具体失败原因。
- 修复升级后已删除的迁移账号重新出现的问题，包括 Gemini OAuth 账号。
- 删除账号导入弹窗中关于“分组需手工绑定”的旧提示。

### 测试

- 补充 CPA/Codex、ZIP、兼容性模式、多文件、多链接、批量设置和迁移跳过相关测试。
- 新增错误信息 i18n 相关测试：语言解析与 `Accept-Language` 协商、消息翻译与回退，以及错误出口在 `Accept-Language: zh` 下返回中文消息体的端到端测试。

## v0.1.3

### 新增

- 新增自定义 UI 主题/板块系统，支持管理员通过 ZIP 上传或 GitHub 仓库导入主题包。
- 支持主题 CSS、配置变量、自定义菜单模块、Markdown 页面板块和 iframe 板块。
- 新增主题激活、停用、配置更新、删除和公开只读主题资源服务。
- 新增 `lightbridge-ui-theme` 终端命令，支持通过 Admin API Key 远程安装、配置、激活、回滚主题。
- 新增 `docs/UI_THEME_AGENT_CONFIG.md`，提供面向 AI agent 的主题包格式、API、CLI 和 curl 配置文档。

### 安全 / 修复

- 限制主题包文件类型、大小和路径，防止路径穿越与危险文件写入。
- 禁止主题 CSS 中的脚本、外部 URL、data URL、expression、behavior、@import 等危险构造。
- 激活主题时自动刷新 embedded frontend HTML 缓存，未激活主题时保持默认前端行为。
