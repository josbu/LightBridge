# 管理员 - 运维监控

本章介绍运维仪表盘和错误分析功能。

---

## 运维仪表盘 `/admin/ops`

实时运维监控仪表盘。

### 控制栏

| 功能 | 操作方式 | 说明 |
|------|---------|------|
| 切换时间范围 | 选择时间范围 | 5m / 30m / 1h / 6h / 24h / 自定义 |
| 按平台筛选 | 选择平台上拉 | 按上游平台过滤 |
| 按分组筛选 | 选择分组下拉 | 按分组 ID 过滤 |
| 查询模式 | 选择查询模式 | auto / raw / preagg |
| 自动刷新 | 开关自动刷新 | 可配置间隔（默认 30 秒） |
| 手动刷新 | 点击刷新按钮 | 立即更新数据 |
| 全屏模式 | 点击全屏按钮 | 进入全屏监控视图 |
| 自定义面板 | 点击「自定义」按钮 | 配置仪表盘卡片的显隐 |
| 设置 | 点击设置按钮 | 打开仪表盘设置弹窗 |
| 告警规则 | 点击告警规则按钮 | 管理告警规则 |
| 查看请求详情 | 点击请求详情按钮 | 打开请求详情浏览器 |
| 查看错误详情 | 点击错误详情按钮 | 打开错误详情浏览器 |

### 监控卡片

| 卡片 | 说明 |
|------|------|
| 可用性网格 | 30 天可用性热力图 |
| 并发指标 | 实时并发数监控 |
| 切换率趋势 | 5 小时窗口模型切换率趋势 |
| 吞吐量趋势 | 吞吐量趋势图，支持按平台/分组分解 |
| 延迟分布 | 延迟直方图 |
| 错误分布 | 按类型分布的错误统计 |
| 错误趋势 | 错误趋势折线图 |
| OpenAI Token 统计 | OpenAI 专用 Token 使用统计 |
| 告警事件 | 告警事件历史 |
| 系统日志 | 系统日志条目 |

## 错误分析 `/admin/error-analysis`

深度错误分析和诊断工具。

### 搜索与筛选

| 功能 | 操作方式 | 说明 |
|------|---------|------|
| 搜索错误 | 输入搜索关键词 | 自由文本搜索（350ms 防抖） |
| 按用户筛选 | 输入用户邮箱或 ID | 支持数字 ID 和邮箱 |
| 按时间范围筛选 | 选择起止时间 | 默认最近 24 小时 |
| 按状态码筛选 | 选择状态下拉 | All / 403 / 429 / 500 / 502 / 503 / 504 |
| 按已读状态筛选 | 选择已读下拉 | All / Unread / Read |
| 导出所有错误 | 点击导出按钮 | 导出为 TXT 文件（含详细分析） |
| 清除错误 | 点击清除按钮 | 批量删除所有匹配错误（需确认） |

### 错误详情分析面板

点击错误列表项后，右侧显示详细分析：

| 区域 | 内容 |
|------|------|
| 头部信息 | 状态码、阶段、错误归属、根因标题、根模块、置信度 |
| 证据网格 | 2x2 证据卡片，按颜色编码（好/警告/危险） |
| 步骤流程 | 7 步流水线可视化，每步显示状态（通过/失败/警告/跳过） |
| 步骤详情 | 每步的详细证据卡片和状态标签 |
| 账号诊断 | 调度账号的可用性、容量、阻塞原因 |
| 原始响应 | 可切换的 JSON 响应体 |
| 建议列表 | 基于分析的操作建议 |
| 上游尝试 | 关联的上游错误（状态码、账号名、错误消息） |

查看详情后自动标记为已读。
